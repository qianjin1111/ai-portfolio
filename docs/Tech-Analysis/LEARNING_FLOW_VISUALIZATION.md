## 代码分析流程

```
┌──────────────────────────────────────────────────────────────────────┐
│                      代码分析优先级策略                               │
└──────────────────────────────────────────────────────────────────────┘

                    P0: 必读文件
┌──────────────────────────────────────────────────────────────────────┐
│  README.md          package.json         src/index.ts/main.py       │
│  (项目介绍)          (依赖配置)          (入口文件)                  │
│  docs/*.md          setup.py                                      │
│  (官方文档)          (依赖配置)                                     │
└──────────────────────────────────────────────────────────────────────┘
                           │
                           ▼
                    P1: 重要文件
┌──────────────────────────────────────────────────────────────────────┐
│  src/core/*          src/services/*         lib/*                    │
│  (核心模块)          (服务层)              (库文件)                  │
│                                                                      │
│  tests/*            test/*                                         │
│  (测试用例)          (测试用例)                                     │
└──────────────────────────────────────────────────────────────────────┘
                           │
                           ▼
                    P2: 辅助文件
┌──────────────────────────────────────────────────────────────────────┐
│  examples/*         config/*              .github/*                  │
│  (示例代码)          (配置文件)            (CI/CD配置)                │
└──────────────────────────────────────────────────────────────────────┘

                           │
                           ▼
                    代码阅读顺序
┌──────────────────────────────────────────────────────────────────────┐
│                                                                      │
│  1. 宏观理解: 目录结构 → 模块划分                                    │
│                                                                      │
│  2. 入口追踪: 入口文件 → 主要类/函数 → 调用链                         │
│                                                                      │
│  3. 核心深入: 核心算法 → 逐步解析 → 理解设计                         │
│                                                                      │
│  4. 细节补充: 工具函数 → 辅助模块 → 数据结构                         │
│                                                                      │
│  5. 配置理解: 配置文件 → 环境变量 → 默认值                           │
│                                                                      │
└──────────────────────────────────────────────────────────────────────┘
```

```

│                    技术博客文章撰写流程                               │

                    1. 项目背景与价值分析
┌──────────────────────────────────────────────────────────────────────┐
│  • 项目定位与核心价值                                               │

                           │

┌──────────────────────────────────────────────────────────────────────┐

                           │
                           ▼
                    3. 关键技术实现剖析
┌──────────────────────────────────────────────────────────────────────┐

│  • 关键算法（代码片段 + 复杂度分析）                                │
│  • 集成方案（代码片段 + 流程说明）                                  │

                           │

│  • 改进方案（4个具体建议，带代码示例）                             │
│  • 技术路线建议                                                   │

                           │
                           ▼
```
